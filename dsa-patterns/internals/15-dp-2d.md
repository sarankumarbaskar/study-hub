# 2D Dynamic Programming — Interview Execution Playbook

> **Pattern Mastery Level:** 2D DP is the backbone of FAANG hard-tier interviews. It appears in ~20% of DP problems, and interviewers expect you to define the state, write the recurrence, and offer a space optimization — all within 20 minutes. Three flavors: grid paths, two-string comparison, and knapsack.

---

## 1. Pattern Recognition Signals

### When to Use 2D DP

```
INSTANT TRIGGERS (say "2D DP" within 10 seconds):
  ✓ Grid + "count paths" or "minimum cost path" (right/down moves)
  ✓ Two strings + "longest common", "edit distance", "interleaving"
  ✓ Array of items + "target sum" or "capacity" (knapsack)
  ✓ "How many ways to reach / form / partition"
  ✓ One string + "longest palindromic subsequence" (trick: compare string with its reverse)
```

### Keywords in Problem Statements

```
GRID DP:                 TWO-SEQUENCE DP:           KNAPSACK DP:
  "grid of values"         "longest common"           "target sum"
  "top-left to             "edit distance"            "coin change"
   bottom-right"           "interleaving"             "subset sum"
  "only move right         "match pattern"            "knapsack"
   or down"                "transform X to Y"         "number of ways
  "obstacles"              "subsequence of both"         to make amount"
  "minimum path sum"       "minimum operations"       "partition into
                                                        two subsets"
```

### When NOT to Use

```
✗ Single sequence optimization → 1D DP (climbing stairs, house robber)
✗ Problem has optimal substructure on intervals → Interval DP (burst balloons)
✗ Problem needs all permutations/combinations → Backtracking
✗ Greedy choice property holds → Greedy (activity selection)
✗ State needs more than two dimensions → consider bitmask DP or 3D DP
```

## 2. Thinking Framework (Step-by-Step Intuition)

### The 90-Second Decision Process

```
Step 1: "What are my TWO dimensions?"
  Grid?           → rows and columns: dp[r][c]
  Two sequences?  → prefix lengths: dp[i][j] for first i of X, first j of Y
  Items + target? → first i items, capacity/sum j: dp[i][j]

Step 2: "What does dp[i][j] MEAN?"
  Write it in plain English BEFORE coding:
    Grid:     "min cost to reach cell (i, j)"
    LCS:      "length of LCS for text1[0..i) and text2[0..j)"
    Edit:     "min ops to convert word1[0..i) into word2[0..j)"
    Knapsack: "number of ways to make sum j using first i items"

Step 3: "What are the BASE CASES?"
  Grid:     first row and first column (only one direction to arrive)
  Strings:  dp[0][j] and dp[i][0] — one or both strings are empty
  Knapsack: dp[i][0] = 1 (one way to make sum 0: pick nothing)

Step 4: "What is the RECURRENCE?"
  At each cell, what choices do I have?
    Grid:     come from above OR from left
    Match:    chars equal → diagonal; else → min/max of three neighbors
    Knapsack: skip item OR take item (adjust remaining capacity)

Step 5: "Can I SPACE OPTIMIZE?"
  If dp[i][j] only depends on dp[i-1][...] and dp[i][...]:
    → keep only two rows (prev[] and curr[]) → O(n) space
  If dp[i][j] depends on dp[i-1][j-1], dp[i-1][j], dp[i][j-1]:
    → one row + one variable for the diagonal → O(n) space
```

### The Core Insight (Memorize This)

```
2D DP WORKS BECAUSE:
  The answer for the FULL problem dp[m][n] is built from answers to
  SMALLER subproblems dp[i][j] where i ≤ m and j ≤ n.

  Fill order is ROW BY ROW, LEFT TO RIGHT. When you compute dp[i][j],
  everything it depends on — dp[i-1][j], dp[i][j-1], dp[i-1][j-1] —
  has already been computed.

  Space optimization: since each row only needs the PREVIOUS row,
  collapse the 2D table into a 1D array and overwrite in-place.
```

### Three Flavor Cheat Sheet

```
FLAVOR          STATE dp[i][j]                    RECURRENCE CORE
────────────────────────────────────────────────────────────────────────
Grid Path       ways/cost to reach (i,j)          dp[i-1][j] ○ dp[i][j-1]
Two-Sequence    answer for X[0..i), Y[0..j)       match → diag; else → 3 neighbors
Knapsack        answer using first i items, cap j  skip: dp[i-1][j]; take: dp[...][j-w]

○ = + for counting, min() for cost, max() for value
```

## 3. Java Templates (Production-Quality)

### Template 1: Grid Path DP

```java
// USE FOR: Unique Paths (#62), Minimum Path Sum (#64), Unique Paths II (#63)
// TIME: O(m·n) | SPACE: O(m·n), optimizable to O(n)
public int gridDP(int[][] grid) {
    int m = grid.length, n = grid[0].length;
    int[][] dp = new int[m][n];
    dp[0][0] = grid[0][0];
    for (int c = 1; c < n; c++) dp[0][c] = dp[0][c - 1] + grid[0][c];
    for (int r = 1; r < m; r++) dp[r][0] = dp[r - 1][0] + grid[r][0];
    for (int r = 1; r < m; r++)
        for (int c = 1; c < n; c++)
            dp[r][c] = Math.min(dp[r - 1][c], dp[r][c - 1]) + grid[r][c];
    return dp[m - 1][n - 1];
}
```

### Template 2: LCS (Two-Sequence)

```java
// USE FOR: Longest Common Subsequence (#1143), Longest Palindromic Subsequence (#516)
// TIME: O(m·n) | SPACE: O(m·n), optimizable to O(n)
public int lcs(String s1, String s2) {
    int m = s1.length(), n = s2.length();
    int[][] dp = new int[m + 1][n + 1];
    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++)
            if (s1.charAt(i - 1) == s2.charAt(j - 1))
                dp[i][j] = dp[i - 1][j - 1] + 1;
            else
                dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
    return dp[m][n];
}
```

### Template 3: Edit Distance (Two-Sequence with Three Operations)

```java
// USE FOR: Edit Distance (#72), any min-operations-to-transform problem
// TIME: O(m·n) | SPACE: O(m·n), optimizable to O(n)
public int editDistance(String word1, String word2) {
    int m = word1.length(), n = word2.length();
    int[][] dp = new int[m + 1][n + 1];
    for (int i = 0; i <= m; i++) dp[i][0] = i;
    for (int j = 0; j <= n; j++) dp[0][j] = j;
    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++)
            if (word1.charAt(i - 1) == word2.charAt(j - 1))
                dp[i][j] = dp[i - 1][j - 1];
            else
                dp[i][j] = 1 + Math.min(dp[i - 1][j - 1],
                        Math.min(dp[i - 1][j], dp[i][j - 1]));
    return dp[m][n];
}
```

### Template 4: 0/1 Knapsack (Bounded — Each Item Used Once)

```java
// USE FOR: Target Sum (#494), Partition Equal Subset Sum (#416), subset sum variants
// TIME: O(n·W) | SPACE: O(n·W), optimizable to O(W)
public int knapsack01(int[] items, int capacity) {
    int n = items.length;
    int[][] dp = new int[n + 1][capacity + 1];
    dp[0][0] = 1; // one way to make sum 0
    for (int i = 0; i < n; i++)
        for (int j = 0; j <= capacity; j++) {
            dp[i + 1][j] = dp[i][j]; // skip
            if (j >= items[i])
                dp[i + 1][j] += dp[i][j - items[i]]; // take (previous row = 0/1)
        }
    return dp[n][capacity];
}
```

### Template 5: Unbounded Knapsack (Each Item Reusable)

```java
// USE FOR: Coin Change 2 (#518), unbounded knapsack variants
// TIME: O(n·W) | SPACE: O(n·W), optimizable to O(W)
public int knapsackUnbounded(int[] items, int capacity) {
    int n = items.length;
    int[][] dp = new int[n + 1][capacity + 1];
    for (int i = 0; i <= n; i++) dp[i][0] = 1;
    for (int i = 0; i < n; i++)
        for (int j = 0; j <= capacity; j++) {
            dp[i + 1][j] = dp[i][j]; // skip
            if (j >= items[i])
                dp[i + 1][j] += dp[i + 1][j - items[i]]; // take (SAME row = unbounded)
        }
    return dp[n][capacity];
}
```

### Template 6: Space-Optimized 2D → 1D

```java
// GRID DP (Unique Paths) — two rows → one row
public int uniquePathsOptimized(int m, int n) {
    int[] dp = new int[n];
    Arrays.fill(dp, 1);
    for (int r = 1; r < m; r++)
        for (int c = 1; c < n; c++)
            dp[c] += dp[c - 1]; // dp[c] already holds the "above" value
    return dp[n - 1];
}

// TWO-SEQUENCE DP (LCS) — 2D → 1D + diagonal variable
public int lcsOptimized(String s1, String s2) {
    int m = s1.length(), n = s2.length();
    int[] dp = new int[n + 1];
    for (int i = 1; i <= m; i++) {
        int prev = 0; // holds dp[i-1][j-1]
        for (int j = 1; j <= n; j++) {
            int temp = dp[j]; // save dp[i-1][j] before overwrite
            if (s1.charAt(i - 1) == s2.charAt(j - 1))
                dp[j] = prev + 1;
            else
                dp[j] = Math.max(dp[j], dp[j - 1]);
            prev = temp;
        }
    }
    return dp[n];
}

// 0/1 KNAPSACK — 2D → 1D (iterate capacity BACKWARDS)
public int knapsack01Optimized(int[] items, int capacity) {
    int[] dp = new int[capacity + 1];
    dp[0] = 1;
    for (int item : items)
        for (int j = capacity; j >= item; j--) // BACKWARDS prevents reuse
            dp[j] += dp[j - item];
    return dp[capacity];
}

// UNBOUNDED KNAPSACK — 2D → 1D (iterate capacity FORWARDS)
public int knapsackUnboundedOptimized(int[] items, int capacity) {
    int[] dp = new int[capacity + 1];
    dp[0] = 1;
    for (int item : items)
        for (int j = item; j <= capacity; j++) // FORWARDS allows reuse
            dp[j] += dp[j - item];
    return dp[capacity];
}
```

## 4. Edge Case Checklist

```
GRID DP:
  □ 1×1 grid → answer is the single cell value
  □ 1×n or m×1 grid → only one path (straight line)
  □ Obstacle at (0,0) or (m-1,n-1) → immediately return 0
  □ Obstacle blocks entire first row/column → all cells after it are unreachable
  □ Grid values can be 0 → doesn't block path (only obstacles do)
  □ Large grids → integer overflow in path counting (use long if m,n > 30)

TWO-SEQUENCE DP:
  □ One or both strings empty → base case drives answer (edit distance = length of other)
  □ Strings are identical → LCS = length, edit distance = 0
  □ Single character strings → check match/mismatch
  □ 1-indexed dp but 0-indexed strings → dp[i][j] uses charAt(i-1), charAt(j-1)
  □ Palindromic subsequence → compare s with reverse(s), it's just LCS

KNAPSACK DP:
  □ Target = 0 → usually 1 way (take nothing)
  □ All items larger than capacity → answer is base case (0 ways or impossible)
  □ Item with value 0 → doubles the count for every subset (2^count_of_zeros multiplier)
  □ (sum + target) is odd or negative → no valid partition exists for Target Sum
  □ Negative numbers in items → offset needed or use HashMap-based DP

GENERAL:
  □ dp array size is (m+1)×(n+1) for string problems (row/col 0 = empty prefix)
  □ dp array size is m×n for grid problems (direct cell mapping)
  □ Space optimization loses the ability to reconstruct the actual solution path
```

#### [Unique Paths](https://leetcode.com/problems/unique-paths/) — LeetCode #62

- **Brute Force:** DFS/backtrack exploring every right/down path. Time O(2^(m+n)); Space O(m+n).
- **Intuition:** Robot moves only right or down. `dp[r][c]` = number of ways to reach `(r,c)`. Each cell receives paths from above and left. First row and column have exactly 1 path each (straight line).
- **Approach:** Initialize first row/column to 1. Fill table row by row.

```java
class Solution {
    public int uniquePaths(int m, int n) {
        int[] dp = new int[n];
        Arrays.fill(dp, 1);
        for (int r = 1; r < m; r++)
            for (int c = 1; c < n; c++)
                dp[c] += dp[c - 1];
        return dp[n - 1];
    }
}
```

- **Complexity:** Time O(m·n) — fill every cell once; Space O(n) — single row.

#### [Minimum Path Sum](https://leetcode.com/problems/minimum-path-sum/) — LeetCode #64

- **Brute Force:** Try all paths from (0,0) to (m-1,n-1), sum each, take minimum. Time O(2^(m+n)); Space O(m+n).
- **Intuition:** `dp[r][c]` = minimum sum to reach `(r,c)`. Either come from above or left. Base: first row and column are cumulative sums (only one direction).

```java
class Solution {
    public int minPathSum(int[][] grid) {
        int m = grid.length, n = grid[0].length;
        int[][] dp = new int[m][n];
        dp[0][0] = grid[0][0];
        for (int c = 1; c < n; c++) dp[0][c] = dp[0][c - 1] + grid[0][c];
        for (int r = 1; r < m; r++) dp[r][0] = dp[r - 1][0] + grid[r][0];
        for (int r = 1; r < m; r++)
            for (int c = 1; c < n; c++)
                dp[r][c] = grid[r][c] + Math.min(dp[r - 1][c], dp[r][c - 1]);
        return dp[m - 1][n - 1];
    }
}
```

- **Complexity:** Time O(m·n); Space O(m·n) — can optimize to O(n) with a single row.

#### [Longest Common Subsequence](https://leetcode.com/problems/longest-common-subsequence/) — LeetCode #1143

- **Brute Force:** Generate all 2^m subsequences of text1, check each against text2. Time O(2^m·n); Space O(m).
- **Intuition:** `dp[i][j]` = LCS length of `text1[0..i)` and `text2[0..j)`. If chars match, extend the diagonal by 1. If not, take the best of skipping either character.

```java
class Solution {
    public int longestCommonSubsequence(String text1, String text2) {
        int m = text1.length(), n = text2.length();
        int[] dp = new int[n + 1];
        for (int i = 1; i <= m; i++) {
            int prev = 0;
            for (int j = 1; j <= n; j++) {
                int temp = dp[j];
                if (text1.charAt(i - 1) == text2.charAt(j - 1))
                    dp[j] = prev + 1;
                else
                    dp[j] = Math.max(dp[j], dp[j - 1]);
                prev = temp;
            }
        }
        return dp[n];
    }
}
```

- **Complexity:** Time O(m·n); Space O(n) — space-optimized single row.

#### [Edit Distance](https://leetcode.com/problems/edit-distance/) — LeetCode #72

- **Brute Force:** Recursively try insert/delete/replace at every mismatch. Time O(3^(m+n)); Space O(m+n).
- **Intuition:** `dp[i][j]` = min operations to convert `word1[0..i)` into `word2[0..j)`. If chars match: free (diagonal). Else: 1 + min of three operations — insert (`dp[i][j-1]`), delete (`dp[i-1][j]`), replace (`dp[i-1][j-1]`).
- **Base cases:** `dp[i][0] = i` (delete all), `dp[0][j] = j` (insert all).

```java
class Solution {
    public int minDistance(String word1, String word2) {
        int m = word1.length(), n = word2.length();
        int[][] dp = new int[m + 1][n + 1];
        for (int i = 0; i <= m; i++) dp[i][0] = i;
        for (int j = 0; j <= n; j++) dp[0][j] = j;
        for (int i = 1; i <= m; i++)
            for (int j = 1; j <= n; j++)
                if (word1.charAt(i - 1) == word2.charAt(j - 1))
                    dp[i][j] = dp[i - 1][j - 1];
                else
                    dp[i][j] = 1 + Math.min(dp[i - 1][j - 1],
                            Math.min(dp[i - 1][j], dp[i][j - 1]));
        return dp[m][n];
    }
}
```

- **Complexity:** Time O(m·n); Space O(m·n) — can optimize to O(n) with single row + diagonal variable.

#### [Longest Palindromic Subsequence](https://leetcode.com/problems/longest-palindromic-subsequence/) — LeetCode #516

- **Brute Force:** Generate all 2^n subsequences, check each for palindrome, take max. Time O(2^n·n); Space O(n).
- **Intuition:** The longest palindromic subsequence of `s` is the LCS of `s` and `reverse(s)`. This directly reduces to the LCS template — no new pattern needed.

```java
class Solution {
    public int longestPalindromeSubseq(String s) {
        String rev = new StringBuilder(s).reverse().toString();
        int n = s.length();
        int[] dp = new int[n + 1];
        for (int i = 1; i <= n; i++) {
            int prev = 0;
            for (int j = 1; j <= n; j++) {
                int temp = dp[j];
                if (s.charAt(i - 1) == rev.charAt(j - 1))
                    dp[j] = prev + 1;
                else
                    dp[j] = Math.max(dp[j], dp[j - 1]);
                prev = temp;
            }
        }
        return dp[n];
    }
}
```

- **Complexity:** Time O(n²); Space O(n) — space-optimized LCS on string and its reverse.

#### [Target Sum](https://leetcode.com/problems/target-sum/) — LeetCode #494

- **Brute Force:** Try all 2^n assignments of + or - to each number. Time O(2^n); Space O(n).
- **Intuition:** Assign + or - to each number. Let P = subset with +, S = total sum. We need `P - (S - P) = target`, so `P = (S + target) / 2`. If `(S + target)` is odd or negative, return 0. This reduces to **0/1 knapsack count**: how many subsets sum to `(S + target) / 2`.
- **Key insight:** 0/1 means each item used at most once → iterate capacity **backwards** in 1D optimization.

```java
class Solution {
    public int findTargetSumWays(int[] nums, int target) {
        int sum = 0;
        for (int x : nums) sum += x;
        int diff = sum + target;
        if (diff < 0 || diff % 2 != 0) return 0;
        int t = diff / 2;
        int[] dp = new int[t + 1];
        dp[0] = 1;
        for (int num : nums)
            for (int j = t; j >= num; j--) // BACKWARDS for 0/1
                dp[j] += dp[j - num];
        return dp[t];
    }
}
```

- **Complexity:** Time O(n·t) where t = (sum + target) / 2; Space O(t).

#### [Coin Change 2](https://leetcode.com/problems/coin-change-ii/) — LeetCode #518

- **Brute Force:** Recursively try all combinations of coins. Time O(amount^n); Space O(amount).
- **Intuition:** Count combinations (not permutations) to make `amount`. Each coin is **unbounded** (reusable). `dp[i][j]` = ways to make `j` using first `i` coin types. Skip coin: `dp[i-1][j]`. Use coin: `dp[i][j - coin]` (same row because unbounded).
- **Key insight:** Unbounded → iterate capacity **forwards** in 1D. Outer loop over coins (not amounts) to avoid counting permutations.

```java
class Solution {
    public int change(int amount, int[] coins) {
        int[] dp = new int[amount + 1];
        dp[0] = 1;
        for (int coin : coins)
            for (int j = coin; j <= amount; j++) // FORWARDS for unbounded
                dp[j] += dp[j - coin];
        return dp[amount];
    }
}
```

- **Complexity:** Time O(n·amount); Space O(amount).

#### [Interleaving String](https://leetcode.com/problems/interleaving-string/) — LeetCode #97

- **Brute Force:** Try all interleavings via recursion (pick from s1 or s2 at each step). Time O(2^(m+n)); Space O(m+n).
- **Intuition:** `dp[i][j]` = can `s3[0..i+j)` be formed by interleaving `s1[0..i)` and `s2[0..j)`? At position `i+j-1` in s3, the character came from either s1 or s2. Check both and OR the results.
- **Base:** `dp[0][0] = true`. First row: s2 alone matches s3 prefix. First column: s1 alone matches.
- **Early exit:** If `len(s1) + len(s2) != len(s3)`, return false immediately.

```java
class Solution {
    public boolean isInterleave(String s1, String s2, String s3) {
        int m = s1.length(), n = s2.length();
        if (m + n != s3.length()) return false;
        boolean[] dp = new boolean[n + 1];
        for (int i = 0; i <= m; i++)
            for (int j = 0; j <= n; j++) {
                if (i == 0 && j == 0) {
                    dp[j] = true;
                } else if (i == 0) {
                    dp[j] = dp[j - 1] && s2.charAt(j - 1) == s3.charAt(j - 1);
                } else if (j == 0) {
                    dp[j] = dp[j] && s1.charAt(i - 1) == s3.charAt(i - 1);
                } else {
                    dp[j] = (dp[j] && s1.charAt(i - 1) == s3.charAt(i + j - 1))
                          || (dp[j - 1] && s2.charAt(j - 1) == s3.charAt(i + j - 1));
                }
            }
        return dp[n];
    }
}
```

- **Complexity:** Time O(m·n); Space O(n) — space-optimized single row.

## 6. Common Mistakes & Interview Traps

### Mistake 1: Off-by-One in Two-Sequence DP

```
WRONG: dp[i][j] uses s1.charAt(i) and s2.charAt(j)
  dp is (m+1)×(n+1) with row 0 and col 0 as empty-prefix base cases.
  dp[i][j] corresponds to the first i characters → charAt(i-1), NOT charAt(i).

CORRECT: dp[i][j] → s1.charAt(i-1), s2.charAt(j-1)
  This is the #1 source of bugs in interview DP solutions.
  Say out loud: "dp[i][j] means prefix of LENGTH i, so last char is at INDEX i-1."
```

### Mistake 2: 0/1 vs Unbounded Knapsack Direction

```
CRITICAL DISTINCTION:
  0/1 knapsack (each item once):   iterate j from capacity DOWN to item
    → dp[j] still holds the "previous row" value when we read it
    
  Unbounded knapsack (item reusable): iterate j from item UP to capacity
    → dp[j - item] reflects "same row" = current item already used

WRONG: Using forward iteration for Target Sum (#494) → counts each number multiple times
WRONG: Using backward iteration for Coin Change 2 (#518) → each coin used only once

This is the single most common knapsack bug. If you get a wrong answer,
CHECK THE LOOP DIRECTION FIRST.
```

### Mistake 3: Forgetting Target Sum Validity Check

```
WRONG: Directly computing dp without checking (sum + target)
  If sum + target is negative → target is impossible
  If sum + target is odd → no integer partition exists
  
CORRECT: Check BOTH conditions before building the DP table:
  int diff = sum + target;
  if (diff < 0 || diff % 2 != 0) return 0;
  int t = diff / 2;

Interviewers will test with target > sum or target that forces an odd split.
```

### Mistake 4: Grid Base Case with Obstacles

```
WRONG: Setting dp[0][j] = 1 for all j in first row when obstacles exist
  An obstacle at (0,3) blocks ALL cells (0,4), (0,5), ... in the first row.
  Same for first column: obstacle at (2,0) blocks (3,0), (4,0), ...

CORRECT: Stop propagation at the obstacle:
  for (int j = 1; j < n; j++)
      dp[0][j] = obstacleGrid[0][j] == 1 ? 0 : dp[0][j - 1];
```

### Mistake 5: Space Optimization Destroying the Diagonal

```
WRONG (LCS 1D): dp[j] = dp[j-1] + 1 when chars match
  dp[j] has already been overwritten to the CURRENT row's value.
  You lost dp[i-1][j-1] which you needed for the diagonal.

CORRECT: Save the diagonal BEFORE overwriting:
  int prev = 0;
  for (int j = 1; j <= n; j++) {
      int temp = dp[j];         // save dp[i-1][j]
      if (chars match) dp[j] = prev + 1;  // prev = dp[i-1][j-1]
      else dp[j] = Math.max(dp[j], dp[j - 1]);
      prev = temp;              // update for next iteration
  }
```

### Mistake 6: Coin Change 2 — Combinations vs Permutations

```
WRONG: Outer loop over amounts, inner loop over coins
  for (int j = 1; j <= amount; j++)
      for (int coin : coins)
          dp[j] += dp[j - coin];
  This counts [1,2] and [2,1] as different → PERMUTATIONS (that's #377 Combination Sum IV)

CORRECT: Outer loop over coins, inner loop over amounts
  for (int coin : coins)
      for (int j = coin; j <= amount; j++)
          dp[j] += dp[j - coin];
  Processing one coin at a time prevents reordering → COMBINATIONS

Interviewers WILL ask: "Why does loop order matter?" Have the answer ready.
```

### What Interviewers Actually Look For

```
JUNIOR:    Can write the 2D table, fill it correctly, return dp[m][n]
SENIOR:    Defines state clearly, handles base cases without bugs,
           offers space optimization unprompted
STAFF:     Immediately identifies the sub-pattern (grid/sequence/knapsack),
           explains why the recurrence is correct, discusses 0/1 vs unbounded,
           knows the palindromic subsequence → LCS reduction
```


### How to Explain Your Approach (Script)

```
STEP 1 (30 seconds): State the brute force
  "Brute force is trying all subsequences / all paths / all subsets,
   which is exponential — O(2^n) or O(2^(m+n))."

STEP 2 (45 seconds): Define the state
  "I'll use dp[i][j] to represent [STATE IN PLAIN ENGLISH].
   For example, dp[i][j] = the length of the longest common subsequence
   of the first i characters of text1 and the first j characters of text2."

STEP 3 (30 seconds): State the recurrence
  "If the characters match, I extend the diagonal: dp[i-1][j-1] + 1.
   Otherwise, I take the max of skipping either character."

STEP 4 (15 seconds): Base cases and fill order
  "Base: dp[0][j] = dp[i][0] = 0 (empty prefix). Fill row by row, left to right."

STEP 5: Code (8-12 minutes)

STEP 6 (30 seconds): Offer optimization
  "This is O(m·n) time and space. I can reduce space to O(n) by keeping
   only the previous row, since each cell only depends on the current
   and previous row."
```

### Follow-Up Questions Interviewers Ask

```
Q: "Can you optimize the space?"
A: "Since dp[i][j] only depends on dp[i-1][...] and dp[i][j-1], I can use
    a single 1D array. For LCS/edit distance, I need one extra variable
    to preserve the diagonal dp[i-1][j-1] before it's overwritten."

Q: "How would you reconstruct the actual LCS / edit operations?"
A: "Keep the full 2D table and backtrack from dp[m][n]: if chars matched,
    go diagonal; if dp came from top, go up; if from left, go left.
    Space optimization prevents this — I'd need the full table."

Q: "What's the difference between 0/1 and unbounded knapsack?"
A: "0/1: each item used at most once → in the 2D table, 'take' reads from
    the PREVIOUS row dp[i-1][j-w]. Unbounded: item reusable → 'take' reads
    from the SAME row dp[i][j-w]. In 1D: 0/1 iterates backwards,
    unbounded iterates forwards."

Q: "Can you solve Longest Palindromic Subsequence without reversing?"
A: "Yes — define dp[i][j] as LPS of s[i..j]. If s[i]==s[j], dp[i][j] =
    dp[i+1][j-1] + 2. Else max(dp[i+1][j], dp[i][j-1]). Fill diagonally.
    But the LCS-with-reverse approach is cleaner and reuses the standard template."

Q: "Why does loop order matter for Coin Change 2?"
A: "Outer loop on coins processes one denomination at a time — you can only
    extend combinations that already include this coin. Outer loop on amounts
    would allow mixing coin orders, counting [1,2] and [2,1] separately,
    giving permutations instead of combinations."
```

```
  Unbounded:      iterate j FORWARDS                   (allows item reuse)
```
