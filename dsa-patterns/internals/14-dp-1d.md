# 1D Dynamic Programming — Interview Execution Playbook

> State definition is EVERYTHING. Once you define what dp[i] means, the recurrence writes itself. Master top-down (memo) vs bottom-up (tabulation), then optimize space from O(n) to O(1).

---

## 1. Pattern Recognition Signals

You're looking at 1D DP when you spot these in the problem statement:

| Signal | Example Phrasing |
|--------|-------------------|
| Optimal substructure | "minimum cost to reach the end", "maximum profit" |
| Overlapping subproblems | Recursive solution repeats same calls (Fibonacci-style tree) |
| Counting distinct ways | "how many ways to...", "number of distinct..." |
| Min/max over a sequence | "fewest coins", "longest subsequence", "maximum product" |
| Sequential decisions | "at each step you can...", "take or skip" |
| Feasibility check | "can you form X?", "is it possible to..." |

**The Three Pillars (all must hold):**
1. **Optimal substructure** — optimal solution to the whole contains optimal solutions to subproblems
2. **Overlapping subproblems** — same subproblem computed multiple times in naive recursion
3. **1D state space** — a single index/amount/position parameterizes the state

**Quick Decision Tree:**
```
Can I define dp[i] such that dp[n] is the answer?
  YES → Is dp[i] computable from dp[0..i-1] in O(1) or O(n)?
    YES → 1D DP (this pattern)
  NO → Consider 2D DP, graph DP, or other paradigms
```

## 2. Thinking Framework

Use this 5-step process on EVERY 1D DP problem. Say it out loud in the interview:

### Step 1: Define the State
> "dp[i] represents ___"

This is the hardest and most important step. Common definitions:
- `dp[i]` = answer considering first `i` elements
- `dp[i]` = answer ending at index `i` (must include `nums[i]`)
- `dp[amount]` = answer for target value `amount`

### Step 2: Write the Recurrence
> "dp[i] depends on ___ because ___"

Express dp[i] in terms of smaller subproblems. Ask: "What choices can I make at position i?"

### Step 3: Identify Base Cases
> "dp[0] = ___ because ___"

The recurrence must bottom out somewhere. Think about the smallest meaningful input.

### Step 4: Determine Iteration Order
> "I iterate left-to-right because dp[i] depends on dp[j] where j < i"

Ensure all dependencies are computed before they're needed.

### Step 5: Optimize Space
> "dp[i] only depends on dp[i-1] and dp[i-2], so I can use two variables"

If dp[i] depends on a fixed window of previous values, collapse the array.

**Top-Down vs Bottom-Up Decision:**

| Criterion | Top-Down (Memo) | Bottom-Up (Tabulation) |
|-----------|----------------|----------------------|
| Derivation | Natural from recursion | Requires iteration order |
| Space optimization | Hard (recursion stack) | Easy (rolling variables) |
| Subproblems computed | Only reachable ones | All of them |
| Interview preference | Faster to write, easier to explain | Better for follow-up "optimize space" |
| Stack overflow risk | Yes (deep recursion) | No |

## 3. Java Templates

### Template A: Linear DP (Fibonacci-style)

Use when: dp[i] = f(dp[i-1], dp[i-2]) — fixed lookback window.

```java
public int linearDP(int[] nums) {
    int n = nums.length;
    int[] dp = new int[n];
    dp[0] = /* base case */;
    dp[1] = /* base case */;

    for (int i = 2; i < n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2]; // or min/max
    }
    return dp[n - 1];
}
```

### Template B: Decision DP (Take or Skip)

Use when: at each position, you choose to include the element or not.

```java
public int decisionDP(int[] nums) {
    int n = nums.length;
    int[] dp = new int[n];
    dp[0] = nums[0];
    dp[1] = Math.max(nums[0], nums[1]);

    for (int i = 2; i < n; i++) {
        int take = nums[i] + dp[i - 2];
        int skip = dp[i - 1];
        dp[i] = Math.max(take, skip);
    }
    return dp[n - 1];
}
```

### Template C: State Machine DP

Use when: multiple states at each position (e.g., holding/not-holding, maxProd/minProd).

```java
public int stateMachineDP(int[] nums) {
    int stateA = /* initial for state A */;
    int stateB = /* initial for state B */;
    int result = stateA;

    for (int i = 1; i < nums.length; i++) {
        int newA = /* transition using stateA, stateB, nums[i] */;
        int newB = /* transition using stateA, stateB, nums[i] */;
        stateA = newA;
        stateB = newB;
        result = Math.max(result, stateA);
    }
    return result;
}
```

### Template D: Space-Optimized (Two Variables)

Use when: dp[i] depends only on dp[i-1] and dp[i-2].

```java
public int spaceOptimized(int n) {
    if (n <= 1) return /* base */;
    int prev2 = /* dp[0] */;
    int prev1 = /* dp[1] */;

    for (int i = 2; i <= n; i++) {
        int curr = prev1 + prev2; // recurrence
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

### Template E: Top-Down Memoization

Use when: recursion is more natural, or not all states are reachable.

```java
public int solve(int[] nums) {
    int[] memo = new int[nums.length];
    Arrays.fill(memo, -1);
    return dp(nums, nums.length - 1, memo);
}

private int dp(int[] nums, int i, int[] memo) {
    if (i < 0) return 0; // base case
    if (memo[i] != -1) return memo[i];

    int take = nums[i] + dp(nums, i - 2, memo);
    int skip = dp(nums, i - 1, memo);
    memo[i] = Math.max(take, skip);
    return memo[i];
}
```

## 4. Edge Cases

| Edge Case | How to Handle |
|-----------|---------------|
| Empty input / n=0 | Return 0 or handle before DP loop |
| Single element / n=1 | Often the answer is `nums[0]` itself |
| All zeros | Product subarray → answer is 0 |
| Leading zeros in string | Decode Ways → no valid decoding, return 0 |
| Negative numbers | Product can flip sign; track both max and min |
| Amount unreachable | Coin Change → return -1, use sentinel `amount+1` |
| Integer overflow | Large counts → use `long`; products → check bounds |
| All elements same | LIS → length 1 (strictly increasing) |
| Circular arrangement | House Robber II → split into two linear subproblems |
| dp[i] can legitimately be 0 | Use `Integer.MIN_VALUE` or `null` (Boolean[]) as sentinel |

## 5. Problem Progression

### Problem 1: Climbing Stairs (LC 70) — Easy

**State:** `dp[i]` = number of ways to reach step `i`
**Recurrence:** `dp[i] = dp[i-1] + dp[i-2]` (arrive via 1-step or 2-step)
**Base:** `dp[0] = 1, dp[1] = 1`
**Category:** Linear DP (Fibonacci-style)

```java
class Solution {
    public int climbStairs(int n) {
        if (n <= 1) return 1;
        int prev2 = 1, prev1 = 1;
        for (int i = 2; i <= n; i++) {
            int curr = prev1 + prev2;
            prev2 = prev1;
            prev1 = curr;
        }
        return prev1;
    }
}
```

**Memoization:**

```java
class Solution {
    public int climbStairs(int n) {
        int[] memo = new int[n + 1];
        Arrays.fill(memo, -1);
        return dp(n, memo);
    }

    private int dp(int n, int[] memo) {
        if (n <= 1) return 1;
        if (memo[n] != -1) return memo[n];
        memo[n] = dp(n - 1, memo) + dp(n - 2, memo);
        return memo[n];
    }
}
```

- **Time:** O(n) | **Space:** O(1) optimized, O(n) memo
- **Interview tip:** Start with recursion tree to show overlapping subproblems, then optimize.

### Problem 2: House Robber (LC 198) — Medium

**State:** `dp[i]` = max money robbing from houses `0..i`
**Recurrence:** `dp[i] = max(dp[i-1], dp[i-2] + nums[i])` (skip house i, or rob it)
**Base:** `dp[0] = nums[0], dp[1] = max(nums[0], nums[1])`
**Category:** Decision DP (take/skip)

```java
class Solution {
    public int rob(int[] nums) {
        if (nums.length == 1) return nums[0];
        int prev2 = nums[0], prev1 = Math.max(nums[0], nums[1]);
        for (int i = 2; i < nums.length; i++) {
            int curr = Math.max(prev1, prev2 + nums[i]);
            prev2 = prev1;
            prev1 = curr;
        }
        return prev1;
    }
}
```

**Memoization:**

```java
class Solution {
    public int rob(int[] nums) {
        int[] memo = new int[nums.length];
        Arrays.fill(memo, -1);
        return dp(nums, nums.length - 1, memo);
    }

    private int dp(int[] nums, int i, int[] memo) {
        if (i < 0) return 0;
        if (memo[i] != -1) return memo[i];
        memo[i] = Math.max(dp(nums, i - 1, memo), nums[i] + dp(nums, i - 2, memo));
        return memo[i];
    }
}
```

- **Time:** O(n) | **Space:** O(1) optimized, O(n) memo
- **Interview tip:** This is the canonical "take/skip" template. Everything from stock problems to subset sums builds on this.

### Problem 3: Longest Increasing Subsequence (LC 300) — Medium

**State:** `dp[i]` = length of LIS ending at index `i` (must include `nums[i]`)
**Recurrence:** `dp[i] = 1 + max(dp[j])` for all `j < i` where `nums[j] < nums[i]`
**Base:** `dp[i] = 1` for all `i` (element alone is a subsequence of length 1)
**Category:** Enumerate predecessors

**O(n²) DP:**

```java
class Solution {
    public int lengthOfLIS(int[] nums) {
        int n = nums.length;
        int[] dp = new int[n];
        Arrays.fill(dp, 1);
        int maxLen = 1;

        for (int i = 1; i < n; i++) {
            for (int j = 0; j < i; j++) {
                if (nums[j] < nums[i]) {
                    dp[i] = Math.max(dp[i], dp[j] + 1);
                }
            }
            maxLen = Math.max(maxLen, dp[i]);
        }
        return maxLen;
    }
}
```

**O(n log n) with Binary Search (Patience Sorting):**

```java
class Solution {
    public int lengthOfLIS(int[] nums) {
        // tails[i] = smallest tail element for increasing subsequence of length i+1
        List<Integer> tails = new ArrayList<>();

        for (int num : nums) {
            int pos = Collections.binarySearch(tails, num);
            if (pos < 0) pos = -(pos + 1);
            if (pos == tails.size()) {
                tails.add(num);
            } else {
                tails.set(pos, num);
            }
        }
        return tails.size();
    }
}
```

- **Time:** O(n²) classic / O(n log n) binary search | **Space:** O(n)
- **Interview tip:** Always mention the O(n log n) approach exists. Explain: `tails[i]` stores the smallest possible tail of an increasing subsequence of length `i+1`. For each element, binary search for its position — either extend or replace.

### Problem 4: Word Break (LC 139) — Medium

**State:** `dp[i]` = can string `s[0..i-1]` be segmented into dictionary words?
**Recurrence:** `dp[i] = true` if there exists `j < i` such that `dp[j] == true AND s[j..i-1] ∈ wordDict`
**Base:** `dp[0] = true` (empty string is trivially segmentable)
**Category:** Substring/segment DP

```java
class Solution {
    public boolean wordBreak(String s, List<String> wordDict) {
        Set<String> dict = new HashSet<>(wordDict);
        int n = s.length();
        boolean[] dp = new boolean[n + 1];
        dp[0] = true;

        for (int i = 1; i <= n; i++) {
            for (int j = 0; j < i; j++) {
                if (dp[j] && dict.contains(s.substring(j, i))) {
                    dp[i] = true;
                    break;
                }
            }
        }
        return dp[n];
    }
}
```

**Memoization:**

```java
class Solution {
    public boolean wordBreak(String s, List<String> wordDict) {
        Set<String> dict = new HashSet<>(wordDict);
        Boolean[] memo = new Boolean[s.length()];
        return dp(s, 0, dict, memo);
    }

    private boolean dp(String s, int start, Set<String> dict, Boolean[] memo) {
        if (start == s.length()) return true;
        if (memo[start] != null) return memo[start];

        for (int end = start + 1; end <= s.length(); end++) {
            if (dict.contains(s.substring(start, end)) && dp(s, end, dict, memo)) {
                return memo[start] = true;
            }
        }
        return memo[start] = false;
    }
}
```

- **Time:** O(n² · m) where m = max word length (substring cost) | **Space:** O(n)
- **Interview tip:** Optimization — only try substrings up to max word length in the dictionary. Use `Boolean[]` (object) so `null` means unvisited.

### Problem 5: Coin Change (LC 322) — Medium

**State:** `dp[a]` = minimum coins needed to make amount `a`
**Recurrence:** `dp[a] = 1 + min(dp[a - c])` for each coin `c` where `a >= c`
**Base:** `dp[0] = 0` (zero coins for amount zero)
**Category:** Unbounded choice

```java
class Solution {
    public int coinChange(int[] coins, int amount) {
        int[] dp = new int[amount + 1];
        Arrays.fill(dp, amount + 1); // sentinel "impossible"
        dp[0] = 0;

        for (int a = 1; a <= amount; a++) {
            for (int c : coins) {
                if (a >= c) {
                    dp[a] = Math.min(dp[a], 1 + dp[a - c]);
                }
            }
        }
        return dp[amount] > amount ? -1 : dp[amount];
    }
}
```

**Memoization:**

```java
class Solution {
    public int coinChange(int[] coins, int amount) {
        int[] memo = new int[amount + 1];
        Arrays.fill(memo, -2); // -2 = unvisited
        return dp(coins, amount, memo);
    }

    private int dp(int[] coins, int amount, int[] memo) {
        if (amount == 0) return 0;
        if (amount < 0) return -1;
        if (memo[amount] != -2) return memo[amount];

        int minCoins = Integer.MAX_VALUE;
        for (int c : coins) {
            int sub = dp(coins, amount - c, memo);
            if (sub >= 0) {
                minCoins = Math.min(minCoins, 1 + sub);
            }
        }
        memo[amount] = (minCoins == Integer.MAX_VALUE) ? -1 : minCoins;
        return memo[amount];
    }
}
```

- **Time:** O(amount × coins.length) | **Space:** O(amount)
- **Interview tip:** Use `amount + 1` as infinity (any valid answer ≤ amount). Careful with memo sentinel — since 0 is valid, can't use 0 for "unvisited".

### Problem 6: Maximum Product Subarray (LC 152) — Medium

**State:** Track both `maxProd` and `minProd` ending at current index (negative × negative = positive)
**Recurrence:** At each element, max = max(num, num×prevMax, num×prevMin), min = min(num, num×prevMax, num×prevMin)
**Base:** `maxProd = minProd = nums[0]`
**Category:** State Machine DP (two parallel states)

```java
class Solution {
    public int maxProduct(int[] nums) {
        int maxProd = nums[0], minProd = nums[0], result = nums[0];

        for (int i = 1; i < nums.length; i++) {
            int cur = nums[i];
            int tmpMax = Math.max(cur, Math.max(maxProd * cur, minProd * cur));
            int tmpMin = Math.min(cur, Math.min(maxProd * cur, minProd * cur));
            maxProd = tmpMax;
            minProd = tmpMin;
            result = Math.max(result, maxProd);
        }
        return result;
    }
}
```

- **Time:** O(n) | **Space:** O(1)
- **Interview tip:** Key insight is that negatives flip sign. You MUST track min because min × negative = new max. Compute both new values before overwriting.

### Problem 7: Decode Ways (LC 91) — Medium

**State:** `dp[i]` = number of ways to decode `s[0..i-1]`
**Recurrence:** If `s[i-1] != '0'`: `dp[i] += dp[i-1]` (single digit). If `s[i-2..i-1]` forms 10-26: `dp[i] += dp[i-2]` (two digits).
**Base:** `dp[0] = 1` (empty prefix = one way)
**Category:** Linear DP with conditional transitions

```java
class Solution {
    public int numDecodings(String s) {
        if (s.charAt(0) == '0') return 0;
        int n = s.length();
        int prev2 = 1, prev1 = 1; // dp[0]=1, dp[1]=1 (first char nonzero)

        for (int i = 2; i <= n; i++) {
            int curr = 0;
            int oneDigit = s.charAt(i - 1) - '0';
            int twoDigit = (s.charAt(i - 2) - '0') * 10 + oneDigit;
            if (oneDigit >= 1) curr += prev1;
            if (twoDigit >= 10 && twoDigit <= 26) curr += prev2;
            prev2 = prev1;
            prev1 = curr;
        }
        return prev1;
    }
}
```

**Memoization:**

```java
class Solution {
    public int numDecodings(String s) {
        int[] memo = new int[s.length()];
        Arrays.fill(memo, -1);
        return dp(s, 0, memo);
    }

    private int dp(String s, int i, int[] memo) {
        if (i == s.length()) return 1;
        if (s.charAt(i) == '0') return 0;
        if (memo[i] != -1) return memo[i];

        int ways = dp(s, i + 1, memo);
        if (i + 1 < s.length()) {
            int twoDigit = (s.charAt(i) - '0') * 10 + (s.charAt(i + 1) - '0');
            if (twoDigit <= 26) ways += dp(s, i + 2, memo);
        }
        return memo[i] = ways;
    }
}
```

- **Time:** O(n) | **Space:** O(1) optimized, O(n) memo
- **Interview tip:** The '0' handling is where most people trip up. '0' alone is invalid. Only '10' and '20' use zero in a valid two-digit code.

### Problem 8: Longest Valid Parentheses (LC 32) — Hard

**State:** `dp[i]` = length of longest valid parentheses substring ending at index `i`
**Recurrence:**
- If `s[i] == ')'` and `s[i-1] == '('`: `dp[i] = dp[i-2] + 2`
- If `s[i] == ')'` and `s[i-1] == ')'`: check if `s[i - dp[i-1] - 1] == '('`, then `dp[i] = dp[i-1] + 2 + dp[i - dp[i-1] - 2]`

**Base:** `dp[i] = 0` for all (only ')' positions can have nonzero values)
**Category:** Complex conditional DP

```java
class Solution {
    public int longestValidParentheses(String s) {
        int n = s.length();
        if (n < 2) return 0;
        int[] dp = new int[n];
        int maxLen = 0;

        for (int i = 1; i < n; i++) {
            if (s.charAt(i) == ')') {
                if (s.charAt(i - 1) == '(') {
                    dp[i] = (i >= 2 ? dp[i - 2] : 0) + 2;
                } else if (dp[i - 1] > 0) {
                    int matchIdx = i - dp[i - 1] - 1;
                    if (matchIdx >= 0 && s.charAt(matchIdx) == '(') {
                        dp[i] = dp[i - 1] + 2 + (matchIdx >= 1 ? dp[matchIdx - 1] : 0);
                    }
                }
                maxLen = Math.max(maxLen, dp[i]);
            }
        }
        return maxLen;
    }
}
```

**Stack-based Alternative (cleaner to explain):**

```java
class Solution {
    public int longestValidParentheses(String s) {
        Deque<Integer> stack = new ArrayDeque<>();
        stack.push(-1); // sentinel for "last unmatched" boundary
        int maxLen = 0;

        for (int i = 0; i < s.length(); i++) {
            if (s.charAt(i) == '(') {
                stack.push(i);
            } else {
                stack.pop();
                if (stack.isEmpty()) {
                    stack.push(i); // new boundary
                } else {
                    maxLen = Math.max(maxLen, i - stack.peek());
                }
            }
        }
        return maxLen;
    }
}
```

- **Time:** O(n) | **Space:** O(n)
- **Interview tip:** The DP approach is tricky to derive live. The stack approach is often easier to explain: push indices, and the current valid length = `i - stack.peek()` after popping a match.

## 6. Common Mistakes

### Mistake 1: Wrong Base Case
```java
// WRONG: dp[0] = 0 when it should be 1
dp[0] = 0; // "zero ways to decode empty string" — actually it's 1!
```
**Fix:** Think about what the empty/trivial case means. "How many ways to reach step 0?" → 1 (you're already there).

### Mistake 2: Wrong Iteration Direction
```java
// WRONG: iterating backward when dp[i] needs dp[i-1]
for (int i = n - 1; i >= 0; i--) {
    dp[i] = dp[i + 1] + dp[i + 2]; // dp[i+1] not computed yet!
}
```
**Fix:** Draw the dependency arrow. If dp[i] depends on smaller indices, iterate left-to-right. If it depends on larger indices (suffix problems), iterate right-to-left.

### Mistake 3: Not Considering All Transitions
```java
// WRONG: only checking single-digit decode, forgetting two-digit
dp[i] = dp[i - 1]; // forgot: if s[i-2..i-1] in [10,26], also add dp[i-2]
```
**Fix:** Enumerate ALL choices at position i. In Decode Ways: decode 1 char OR decode 2 chars.

### Mistake 4: Space Optimization Destroys Needed State
```java
// WRONG: optimizing LIS to two variables (dp[i] needs ALL dp[j] for j < i)
int prev1 = 1, prev2 = 1;
// LIS needs dp[j] for arbitrary j < i, cannot collapse!
```
**Fix:** Space optimization only works when dependency is a fixed-size window (i-1, i-2). LIS-style O(n²) needs the full dp array.

### Mistake 5: Sentinel Value Collision
```java
// WRONG: using 0 as "unvisited" in memo when 0 is a valid answer
if (memo[i] != 0) return memo[i]; // 0 is a valid coin count for amount 0!
```
**Fix:** Use `-1`, `-2`, `Integer.MIN_VALUE`, or `Boolean[]`/`Integer[]` with null check.

### Mistake 6: Forgetting to Update Result Separately
```java
// WRONG: assuming dp[n-1] is the answer for LIS
return dp[n - 1]; // LIS answer is max(dp[0..n-1]), not dp[n-1]!
```
**Fix:** When state is "best ending at i", the global answer is `max over all dp[i]`.

## 7. Interview Strategy

### The 4-Minute Framework

| Time | Action |
|------|--------|
| 0:00-0:30 | Read problem, identify DP signals |
| 0:30-1:30 | Define state out loud: "dp[i] represents..." |
| 1:30-2:30 | Write recurrence on paper/whiteboard |
| 2:30-3:00 | Verify with small example (n=3 or 4) |
| 3:00-3:30 | Identify base cases |
| 3:30-4:00 | State time/space complexity, then code |

### What to Say Out Loud

1. **"I see overlapping subproblems because..."** — shows you recognize DP vs greedy
2. **"Let me define dp[i] as..."** — the single most important sentence
3. **"The recurrence is dp[i] = ... because at position i, I can either..."** — shows structured thinking
4. **"Let me verify with a small example"** — trace dp array for input of size 3-4
5. **"I can optimize space because dp[i] only depends on..."** — shows depth of understanding

### Follow-Up Questions to Expect

| Follow-Up | Your Response |
|-----------|---------------|
| "Can you optimize space?" | Collapse to O(1) if fixed lookback window |
| "What about top-down?" | Convert: recursive + memo, same states |
| "What if the array is circular?" | Run linear DP twice on different subarrays |
| "Can you do better than O(n²)?" | LIS → binary search O(n log n); mention technique |
| "What if we need the actual subsequence?" | Track parent pointers / backtrack through dp |

### Pattern Matching Shortcuts

| If problem says... | Think... |
|-------------------|----------|
| "Minimum cost to reach end" | Linear DP, forward computation |
| "Maximum non-adjacent sum" | Take/skip decision DP |
| "Number of ways" | Counting DP, usually addition in recurrence |
| "Can you partition/form/break" | Boolean DP, OR logic |
| "Longest increasing/decreasing" | LIS template, O(n²) or O(n log n) |
| "Minimum coins/stamps" | Unbounded knapsack, iterate over choices |

## 8. Revision + Quick Reference

### One-Line Summaries

| Problem | State | Key Insight | Time |
|---------|-------|-------------|------|
| Climbing Stairs | dp[i] = ways to reach step i | Fibonacci: dp[i] = dp[i-1] + dp[i-2] | O(n) |
| House Robber | dp[i] = max profit from 0..i | Take/skip: max(dp[i-1], dp[i-2]+nums[i]) | O(n) |
| LIS | dp[i] = LIS length ending at i | Check all j < i; binary search for O(n log n) | O(n²) / O(n log n) |
| Word Break | dp[i] = can segment s[0..i-1]? | Try all split points, check dictionary | O(n²·m) |
| Coin Change | dp[a] = min coins for amount a | Try all coins: 1 + dp[a-c] | O(a·k) |
| Max Product | maxP, minP at each index | Negatives flip sign; track both | O(n) |
| Decode Ways | dp[i] = ways to decode s[0..i-1] | 1-digit + 2-digit transitions; handle '0' | O(n) |
| Longest Valid Parens | dp[i] = valid length ending at i | Match ')' with corresponding '(' | O(n) |

### Space Optimization Cheat Sheet

| Can Optimize to O(1)? | Condition | Examples |
|-----------------------|-----------|----------|
| YES | dp[i] depends on dp[i-1], dp[i-2] only | Climbing Stairs, House Robber, Decode Ways |
| YES | Only tracking running max/min | Max Product Subarray |
| NO | dp[i] depends on arbitrary dp[j] for j < i | LIS (O(n²)), Word Break |
| NO | Need full dp array for backtracking | When reconstructing the solution |

### Top-Down vs Bottom-Up Conversion

```
BOTTOM-UP:                          TOP-DOWN:
dp[0] = base                        if (i == 0) return base
for i from 1 to n:                  if (memo[i] != -1) return memo[i]
    dp[i] = f(dp[i-1], dp[i-2])    memo[i] = f(rec(i-1), rec(i-2))
return dp[n]                        return memo[i]
```

They compute the SAME thing. Bottom-up fills all states; top-down only fills reachable states.

### Complexity Reference

| Approach | Time | Space | When |
|----------|------|-------|------|
| Linear DP (tabulation) | O(n) | O(n) → O(1) | Fixed lookback window |
| Quadratic DP | O(n²) | O(n) | Enumerate all predecessors (LIS) |
| DP over amount | O(amount × k) | O(amount) | Unbounded knapsack variant |
| LIS binary search | O(n log n) | O(n) | Optimal LIS when O(n²) TLEs |
| Memoization | Same as tabulation | O(n) + stack | When not all states reachable |
